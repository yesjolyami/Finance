# Монетка

«Монетка» — приложение для учёта денег: расходы, доходы, счета, цели, долги, свои категории, лимиты, статистика и экспорт данных.

## Запуск

```bash
python3 server.py
```

Затем откройте `http://127.0.0.1:3212`.

Данные хранятся в SQLite-файле `finance.db` рядом с проектом. Сервер по умолчанию слушает только `127.0.0.1`, поэтому приложение предназначено для личного локального использования.

Текущий Python-сервер не имеет пользователей, аутентификации, разграничения доступа, CSRF-защиты и встроенного TLS. Не выставляйте его напрямую в интернет и не меняйте `HOST` на `0.0.0.0`, если доступ не закрыт внешним reverse proxy, HTTPS и аутентификацией. Публично сервер раздает только `/`, `/index.html`, `/styles.css` и `/app.js`; база, исходники и служебные файлы по HTTP недоступны.

## Возможности

- добавление доходов и расходов;
- переводы между счетами;
- обычные и накопительные счета для каждого владельца;
- привязка счетов к разным банкам и общий итог по банкам;
- создание, редактирование и удаление счетов;
- фильтр, фактический остаток и ручная установка баланса по каждому счету;
- отдельная история каждого счета с доходами, расходами, переводами и фильтром по месяцам;
- редактирование и удаление операций;
- создание, редактирование и удаление категорий;
- быстрое добавление своей категории прямо из формы операции;
- цели накоплений с прогрессом, датой и пополнениями;
- долги: кто должен, кому должен, остаток и частичные возвраты;
- лимиты по категориям расходов;
- статистика по месяцу, динамика за 6 месяцев, прогноз расходов, топ трат и сравнение с прошлым месяцем;
- переключатель светлой и темной темы;
- экспорт CSV для Excel/Numbers, PNG-отчета и JSON-бэкапа;
- импорт базы из JSON-бэкапа.

Формат текущего JSON-бэкапа зафиксирован в [docs/backup-v5.md](docs/backup-v5.md).

## Проверки

Все серверные тесты используют только временную SQLite-базу:

```bash
python3 -m unittest discover -s tests -p 'test_*.py'
node --check app.js
node --test tests/test_financial_calculations.js
```
